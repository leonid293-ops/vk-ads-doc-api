# Banner

## /api/v2/banners/.json

Ресурс, позволяющий получать, обновлять и удалять рекламные объявления.

Используется объект: [Banner](../object/Banner)

### GET

Получение одного рекламного объявления

Пример запроса

```HTTP

    GET /api/v2/banners/23617841.json
```

Пример ответа

```HTTP

    {
        "id": 23826937
        "ad_group": 9367343
    }

```

### POST

Редактирование рекламного объявления

Пример запроса

```HTTP

    POST /api/v2/banners/23826937.json
    {
        "status": "blocked"
    }
```

Пример ответа

```HTTP

    HTTP 204
```

Содержимое секций urls, textblocks, content полностью замещается при обновлении. К примеру, для баннера с content:

```HTTP

    {
        "content": {
            "logo_image": {
                "id": 83465432
            },
            "promo_image": {
                "id": 90465432
            },
            "tpl_background_image": {
                "id": 86342123
            }
        }
    }
```

запрос обновления: 

```HTTP

    {
        "content": {
            "logo_image": {
                "id": 83465432
            },
            "promo_image": {
                "id": 88278634
            },
            "tpl_background_image_300": {
                "id": 72934723
            }
        }
    }
```

приведет ровно к такому новому содержимому секцию после обновления.

Возможные коды ответа

- 200/204 - объявление сохранено
- 400 - ошибка валидации
- 404 - объявление не найдено

Возможные коды ошибок

- active_banners_limit - превышено количество активных баннеров для пользователя
- deleted_banner - нельзя изменять удаленный баннер
- bad_width - неправильное значение ширины содержимого
- bad_height - неправильное значение высоты содержимого
- bad_size - превышение максимального размера содержимого
- bad_type - неправильный тип содержимого
- bad_length - содержимое имеет недопустимую длительность воспроизведения
- bad_bitrate - содержимое имеет недопустимый битрейт
- dynamic_content - динамический контент использован в недопустимой роли
- custom_params - неправильные параметры видео
- exclamation_signs_limit - превышен лимит на восклицательные знаки в текстовом блоке баннера
- invalid_macros - в текстовом блоке баннера использован неразрешенный макрос
- persistent_urls - в пакете запрещено изменение ссылок
- one_url_object_id - в пакете запрещены ссылки, ведущие на различные объекты
- ad_group_one_url - в пакете запрещены различные ссылки
- url_not_checked - использование не проверенных ссылок запрещено
- url_not_valid - ссылка не подходит указанной роли
- max_value - значение больше максимального
- min_value - значение меньше минимального
- bad_value - неправильный формат или тип значения
- required - поле обязательно
- unallowed_value - значение не входит в список достимых
- bad_items - в списке присутствуют неправильные значения
- unallowed_field - поле недоступно
- read_only_field - обновление неизменяемого поля

В общем случае сообщение о ошибке имеет следующий формат:

```HTTP

    {
        "error": {
            "fields": {
                "": {
                    "message": "",
                    "code": ""
                },
                "": {
                    "message": "",
                    "code": ""
                }
            },
            "message": "Validation failed",
            "code": "validation_failed"
        }
    }
```

где field_name_N - название поля, в котором произошла ошибка, error_message_N - описание ошибки, error_code_2 - код ошибки.

Пример:

```HTTP

    {
        "error": {
            "fields": {
                "call_to_action": {
                    "message": "Unallowed value",
                    "code": "unallowed_value"
                }
            },
            "message": "Validation failed",
            "code": "validation_failed"
        }
    }

```

### DELETE

Удаление рекламного объявления

Пример запроса

```HTTP

    DELETE /api/v2/banners/23617841.json
```

Пример ответа

```HTTP

    HTTP 204
```

Возможные коды ответа

- 204 - объявление удалено
- 404 - объявление не найдено
