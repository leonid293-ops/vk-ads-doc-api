## /api/v2/remarketing/local_geo/1234.json

Ресурс для управления списками локальной географии. После создания списков их можно включить в сегмент, который можно включить в таргетинг кампании.

Используется объект: [LocalGeo](../object/LocalGeo)

### POST

Редактирование списка локальной географии.

Пример запроса:

```HTTP

    POST /api/v2/remarketing/local_geo/1234.json
  {
    "name": "Новое название списка локальной географии",
    "regions": [{
      "lat": 55.75583,
      "lng": 37.6173,
      "radius": 3000,
      "label": "Центр Москвы",
      "address": "Точный адрес",
    }]
  }
```

Пример ответа:

```HTTP

  {
    "id": 1234,
    "name": "Новое название списка локальной географии",
    "regions": [{
      "lat": 55.75583,
      "lng": 37.6173,
      "radius": 3000,
      "label": "Центр Москвы",
      "address": "Точный адрес",
    }]
  }
```

Возвращаемые коды статуса:

200 - если список был успешно изменен.

404 - при попытке изменения несуществующего списка.
Примеры ошибок:

```HTTP

  {"error": {"code": "validation_failed", "fields": { поля, в которых были найдены ошибки }}}

```

### DELETE

Удаление списка локальной географии.
Пример запроса:

```HTTP

   DELETE /api/v2/remarketing/local_geo/1234.json
```

Возвращаемые коды статуса:

204 - Список удален.

404 - Список отсутствует.

409 - Список задействован в ремаркетинговых аудиториях пользователя.89:Tc99,
