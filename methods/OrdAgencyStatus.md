## /api/v2/ord/agency/status.json

### Доступны методы GET и POST.

Ресурс позволяет инициировать отправку, а так же получить статус отправки клиентских актов в ОРД.

### GET

Запрос возвращает список всех статусов отправки актов за указанный месяц _month.

Пример запроса:

```HTTP

   GET/api/v2/ord/agency/status.json?_month=2024-10-01
```

Дополнительные GET-параметры:

С помощью следующих GET-параметров можно фильтровать данные:

- limit - Количество возвращаемых в ответе клиентов. Значение по умолчанию: 20. Максимальное значение: 50.
- offset - Смещение точки отсчета относительно начала списка клиентов. Значение по умолчанию: 0.
- _month - Месяц отчёта.

Поле status может принимать значения:

'new' - новый,  
'uploaded' - загружен,   
'in_queue' - в очереди,  
'in_progress' - в обработке,  
'done' - обработан(успешно отправлены все акты за месяц)  
'failed' - ошибка(см поле error)

```HTTP
{
  "items": [
    {
      "month": "2024-10-09",
      "status": "in_queue",
      "error": "string"
    }
  ]
}
```

### POST

Инициирует отправку актов в ОРД за указанный месяц. Можно выбрать акты каких клиентов отправить, передав параметр _client_id.

```HTTP
{
  "month": "2024-10-09"
}
```

С помощью следующих GET-параметров можно выбрать клиентов для отправки:
- _client_id - только этот клиент.
- _client_id__ne - все кроме этого клиента.
- _client_id__in - только эти.
- _client_id__not_it - все кроме этих.

Пример запроса:

```HTTP

   GET/api/v2/ord/agency/status.json?_client_id__in=12,99
- будут отправлены акты клиентов с id 12 и 99
```

Возвращаемые коды статуса:

200 - Запрос успешно принят.

400 - В следующих случаях:

- Нет актов для отправки (код "no_data").
- Акты агентства уже в процессе передачи в ОРД (код "validation_failed", сообщение 'inconsistent status').
- У агентства есть клиенты, чьи данные были переданы в ОРД с ошибками (код "clients_failed").
