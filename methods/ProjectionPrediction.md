# ProjectionPrediction

## /api/v3/projection.json

Ресурс, возвращающий прогноз охвата аудитории в зависимости от стоимости события.

Используется объект: [ProjectionPrediction](../object/ProjectionPrediction)

### POST

Получение прогноза по кампании или пакетам

Пример запроса

```HTTP

   POST /api/v3/projection.json
   {
       "package_ids": [959, 999],
       "targetings":{
           "pads": [24567, 24568, 33649, 33652]
       }
   }
```

Пример ответа

```HTTP

           {

               "histogram_id": 1,
               "avg_cr": null,

           },
           {
               "package_id": 959,
               "histogram_id": 2,
               "avg_cr": null,
               "avg_ctr": 0.005123368281061198
           }
       ],
       "histograms": [
           {
               "id": 1,
               "histogram": [
                   ...
                   {
                       "price": 52.57,
                       "uniqs": 50589000,
                       "share": 51
                   },
                   {
                       "price": 53.56,
                       "uniqs": 50918000,
                       "share": 51
                   },
                   ...
               ]
           },
           {
               "id": 2,
               "histogram": [
                   ...
                   {
                       "price": 120.48,
                       "uniqs": 72347000,
                       "share": 81
                   },
                   {
                       "price": 121.4,
                       "uniqs": 72420000,
                       "share": 82
                   },
                   ...
               ]
           }
       ]
   }
```

В запросе необходимо указывать либо campaign_id, либо package_ids.

При указании кампании, прогноз будет с учётом её истории.
Существующие таргетинги кампании при построении прогноза не учитываются, необходимо передавать их в запросе.
При указании списка идентификаторов пакетов, будет строиться прогноз для новой кампании в каждом из указанных пакетов.

По ключу targetings можно указать только [таргетинги, учитываемые сервисом прогнозирования](../object/ProjectionTargetings).
Обязательным является только таргетинг pads, возможные значения которого перечислены для каждого из пакетов в package.options.targetings.pads.values.

В targetings.pads можно указывать любые площадки, но прогноз будет строиться только по пересечению присланных площадок с площадками доступными для пакета.
В случае если пересечение будет пустым, в ответе для этого пакета будет пустая гистограмма.

Рекомендуемой эффективной ставкой считается та, которая охватывает 75 и более процентов аудитории.

В прогнозе возвращается информация только о цене за показ (priced_event_type = 0).

Для расчёта стоимости за другие типы событий используйте:

- ``price / (avg_ctr * avg_cr * 1000)`` для события установки (priced_event_type = 7).
- ``price / (avg_ctr * 1000)`` для всех остальных событий.
