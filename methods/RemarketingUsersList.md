## /api/v2/remarketing/users_lists/.json

Ресурс, позволяющий управлять списком пользователей. Такие списки используются для настройки таргетинга на аудиторию, полученную из внешнего источника.

### GET

Возвращает загруженный пользователем список пользователей.

Пример запроса:

```HTTP

  GET /api/v2/remarketing/users_lists/2005000.json
```

Пример ответа:

```HTTP

  {  
    "status":"ready",
    "name":"Лайки и комментарии в ВК",
    "created":"2017-05-29 18:01:05",
    "base":0,
    "entries_count":5001,
    "ids_count":5001,
    "type":"vk",
    "id":2500000,
  }

```

### POST

Запрос изменяет имя списка пользователей.

Пример запроса:

```HTTP

  POST /api/v2/remarketing/users_lists/2005000.json
  {"name": "Новое название списка"}
```

Возвращаемые коды статуса:

204 - Имя успешно изменено.

400 - Ошибка валидации.

404 - Список отсутствует.

Возможные ошибки:

```HTTP

  {"error": {"code": "validation_failed", "fields": {  }}}
 - Ошибка в значениях передаваемых данных
```

### DELETE

Запрос удаляет указанный список  пользователей из доступных источников данных. Удаление возможно только если список не используется ни в одной аудитории и на основе него не создано ни одно lookalike-расширение.

Пример запроса:

```HTTP

  DELETE /api/v2/remarketing/users_lists/2005000.json
```

Возвращаемые коды статуса:

204 - Список успешно удален.

404 - Список отсутствует.

409 - Список задействован в аудиториях или lookalike пользователя.
