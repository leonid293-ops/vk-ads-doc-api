# RemarketingUsersLists

> Оригинал: [https://ads.vk.com/doc/api/resource/RemarketingUsersLists](https://ads.vk.com/doc/api/resource/RemarketingUsersLists)

---

## /api/v2/remarketing/users_lists.json

Ресурс, позволяющий управлять списками пользователей. Такие списки используются для настройки таргетинга на аудиторию пользователей, полученную из внешнего источника.

### GET

Запрос возвращает все загруженные пользователем списки пользователей.

Пример запроса:

```HTTP

  GET /api/v2/remarketing/users_lists.json
```

Пример ответа:

```HTTP

  {  
    "items":[  
      {  
        "status":"ready",
        "name":"Лайки и комментарии в ВК",
        "created":"2017-05-29 18:01:05",
        "base":0,
        "entries_count":5001,
        "ids_count":5001,
        "type":"vk",
        "id":2500000,
      },
      {  
        "status":"ready",
        "name":"Классы и комментарии в ОК",
        "created":"2017-06-16 16:40:54",
        "base":0,
        "entries_count":5001,
        "ids_count":5001,
        "type":"ok",
        "id":2500001,
      }
    ]
  }

```

### POST

Запрос загружает список пользователей в список источников данных для ремаркетинга. В случае успешного добавления возвращается объект типа RemarketingUsersList.

```NOTE
   Список должен содержать не менее 2000 и не более 5000000 строк. Вес файла: не более 128 МБ.
```

Формат файла с типом type=‘ok’:
  ID пользователей, выгруженные из приложения. По одной записи в каждой строке.

Формат файла с типом type=‘dmp_id’:
  Первая строка - код партнера. Остальные строки - ID пользователей.

Формат файла других типов:
  Plain-текстовый список, по одной записи в каждой строке.

Формат CSV:
  CSV-список, в качестве разделителя используется запятая. Для csv-списков с типом type=‘dmp_id’ первая строка - код партнера. Далее — заголовок csv с названиями колонок: id - идентификатор пользователя (обязательное поле) и client_user_id - произвольная строка, которая не должна содержать запятую.

```IMPORTANT
   Тип запроса: multipart/form-data. Сам файл для загрузки нужно передавать как часть multipart-запроса с именем "file". Остальные параметры запроса передаются в JSON-сериализованном виде в параметре "data".
```

Пример запроса:

```HTTP

  POST /api/v2/remarketing/users_lists.json
  Content-Type: multipart/form-data
  file=Файл
  data={"name": "Лайки и комментарии в ВК", "type": "vk"}
```

Пример ответа:

```HTTP

    {  
      "status":"loading",
      "name":"Лайки и комментарии в ВК",
      "created":"2017-05-29 18:01:05",
      "base":0,
      "entries_count":5001,
      "ids_count":5001,
      "type":"vk",
      "id":2500000,
      "history":[  ]
    }
```

Существует возможность добавить или удалить пользователей из уже загруженного списка. Для этого нужно загрузить новый список, указав в параметре "base" одно из следующих значений:
- "" чтобы добавить новых пользователей
- "-" чтобы удалить существующих пользователей
Дифференциальный список должен содержать не менее 25 и не более 5 000 000 строк.

Пример запроса на загрузку дифференциального списка:

```HTTP

  POST /api/v2/remarketing/users_lists.json
  Content-Type: multipart/form-data
  file=Файл
  data={"name": "Лайки и комментарии в ВК", "type": "vk", "base": 10000}
```

Пример ответа:

```HTTP

    {  
      "status":"loading",
      "name":"Лайки и комментарии в ВК",
      "created":"2017-05-29 18:01:05",
      "base":10000,
      "entries_count":30,
      "ids_count":30
      "type":"vk",
      "id":2500000,
      "history":[  ]
    }
```

Возвращаемые коды статуса:

201 - Список успешно загружен.

400 - Ошибка валидации.

403 - Попытка загрузить дифференциальный список со стороны пользователя, не являющегося владельцем базового списка.

Примеры ошибок:

```HTTP

  {"error": {"code": "invalid_json", "message": "Invalid JSON in \"data\" field."}}
 - Некорректный запрос.

  {"error": {"code": "required", "message": "File content is required"}}
 - Некорректно передан файл.

  {"error": {"code": "validation_failed", "fields": {  }}}

 - Ошибка в значениях передаваемых данных.
```
